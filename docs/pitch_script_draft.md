# 댕댕마켓 1페이지 피치 스크립트 (초안)

## 1) 문제
- 반려견 보호자는 **동네에서 믿고 맡길 사람**을 찾기 어렵습니다.
- 기존 커뮤니티는 글은 많아도 **지역 밀도·신뢰 장치**가 약합니다.

## 2) 해결
- 댕댕마켓은 `모이자 · 만나자 · 돌봄 맡기기`를 한 앱에서 연결합니다.
- 핵심은 **하이퍼로컬(구/동 우선 노출)** + **인증 보호맘/댕집사** 흐름입니다.

## 3) 제품 구조
- 커뮤니티: 모이자, 만나자, 그룹채팅.
- 돌봄 연결: 맡기기 글, 방문/맡기기 선택, 거리 기반 매칭.
- 신뢰: 인증 상태, 법적 고지, 운영 정책, RLS 접근제어.

## 4) 왜 지금 가능한가
- `React + Vite + Supabase` 기반으로 빠른 실험/배포가 가능합니다.
- Auth, RLS, Edge Function, 결제 경로가 이미 연결되어 있습니다.

## 5) 수익 모델 (단계형)
- 현재: 런칭 초기 **무료 프로모션 노출**로 공급/수요 밀도 확보.
- 이후: 인증 돌봄/교배 중심 **월 단위 유료 노출** 전환.
- 원칙: 유료 코드 삭제가 아닌 `promo flag + RLS` 전환.

## 6) 차별점
- 전국형이 아닌 **동네 단위**로 먼저 연결 성공률을 올립니다.
- “글 노출”과 “거래 책임”을 분리해 법적 리스크를 관리합니다.

## 7) 요청
- PoC 지역(예: 안양 동안구) 기준으로 사용자/공급자 밀도 실험.
- 지표: 게시글 생성, 채팅 시작률, 돌봄 매칭 전환, 재방문.
