# 댕댕마켓(Dog Meet-up) — 지원사업·사업계획 첨부용 프로젝트 요약

> **문서 목적:** 정부·지자체·액셀러레이터 등에 제출할 때 서비스·기술 내용을 한눈에 전달하기 위한 요약입니다.

---

## 1. 서비스 한 줄 소개

**반려견 커뮤니티·돌봄 연결 웹앱**으로, 모임 글(모이자·만나자), 돌봄 맡기기, **운영자 인증 기반 인증 보호맘** 프로필·예약, Stripe 결제·노출 정책을 **Supabase(PostgreSQL + Auth + RLS + Edge Functions)** 위에 구현한 **PWA(Vite + React)** 서비스입니다.

**사회적 문제의식(프레이밍):** 단순 연결을 넘어, **1인 가구 등에서 반려견 양육·돌봄 공백이 파양으로 이어지는 것을 줄이기 위한 지역 기반 돌봄 품앗이 인프라**, 또는 **반려동물 안전·사고 예방을 위한 운영자 검증을 골자로 한 신뢰 기반 돌봄 네트워크**로도 설명할 수 있습니다(지원사업·발표 시 메시지 선택).

---

## 2. 사회적 필요성 및 문제 인식

| 영역 | 내용 |
|------|------|
| **산책·모임** | 견주 간 지역 기반 만남·소모임 수요는 크나, 신뢰·안전·일정 조율을 한곳에서 다루기 어렵습니다. |
| **돌봄** | 맡기기·맡아주기 정보가 분산되면 검증된 돌봄 제공자를 찾기 어렵습니다. |
| **운영** | **스팸·비인증 돌봄** 리스크를 줄이기 위해 **운영 인증·노출 정책**을 제품에 녹였습니다. |
| **파양·1인 가구** | 출장·질병·시간 부족 등으로 **단기·상호 돌봄**이 없으면 양육 포기 압력이 커질 수 있어, **동네 단위 품앗이·맡김** 채널이 사회적 완충이 될 수 있습니다. |
| **안전·신뢰** | 교상·분실 등 사고 리스크 인식이 커서, **운영자가 기준을 두고 검증한 돌봄 노출**은 단순 중고나라형 나열과 차별화되는 **신뢰 네트워크** 프레임으로 제시 가능합니다. |

본 프로젝트는 **글·프로필·결제·RLS**를 묶어 **운영 가능한 최소 단위 플랫폼**을 목표로 합니다.

---

## 3. 사업·서비스 목표

1. **지역 기반 반려견 모임·정보 교류** 활성화(모이자·만나자·돌봄 카테고리).  
2. **인증 보호맘** — 등록 → 운영자 인증 → 목록 노출(성장 단계에서는 **프로모션 무료 노출** 기본, 유료는 배포·RLS로 단계 전환).  
3. **Stripe** 등으로 유료 노출·예약 결제 **인프라**를 갖추되, **초기에는 공급자 부담을 최소화**해 1~2개 권역에서 점유·신뢰를 우선(상세: `docs/project-pack/08-bm-monetization-strategy.md`).  
4. **모바일 우선**·PWA로 접근성 강화.

## 5. 작동 원리(시스템·데이터)

### 5.1 구조

```
[브라우저 — Vite/React PWA]
  → Supabase Auth(JWT)
  → PostgREST(CRUD + RLS)
  → RPC(예: admin_set_guard_mom_certified)
  → Edge Functions(Stripe Checkout/Webhook, daeng-ai-assist)
[PostgreSQL] RLS·트리거로 인증·노출 필드 보호
```

### 5.2 인증 보호맘

1. 사용자가 `certified_guard_moms`에 프로필 저장.  
2. 관리자가 RPC로 `certified_at` 설정.  
3. SELECT 정책(한시 무료 / 유료 노출)에 따라 목록 공개.  
4. Stripe Webhook으로 노출 기한·예약 연동.

### 5.3 기술 스택

| 계층 | 기술 |
|------|------|
| 프론트 | React 18, React Router 7, Vite, TypeScript, Tailwind CSS, PWA |
| BaaS | Supabase(Auth, DB, RLS, Edge Functions) |
| 결제 | Stripe |

## 6. 차별화·강점

- **운영 인증 + RLS + 결제**를 한 스키마로 묶어 신뢰 가능한 돌봄 노출.  
- **관리자 전용 RPC**로 인증 필드 오조작 완화.  
- **프로모 무료 / 유료** 전환을 마이그레이션·환경변수로 분리.  
- **모바일 우선** UI.

## 7. 향후 과제

- 댕집사 **DB 연동** 목록.  
- 채팅 **실시간·영구 저장** 연동.  
- 개인정보 처리방침·스토어 배포 등 운영 정비.  
- **신뢰·법무:** 교상·분실 등 사고·배상 시 플랫폼 책임 범위는 DB·웹 구현만으로 해결되지 않으며, **이용약관·면책의 정교화**와 **펫 보험 등 제휴·안내(로드맵)** 가 시장 신뢰와 직결됨 — 변호사 검토와 정책 설계를 과제로 둠.
