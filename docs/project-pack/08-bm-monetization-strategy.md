# 08 — 사업 방향성·수익화(BM) 메모

## 외부 평가 요지 (요약)

- **초기 수익 모델 재검토:** 기술적으로 Stripe·유료 노출·RLS를 구현한 것은 플랫폼으로서 갖출 요소이나, **런칭 직후부터 공급자에게 유료 노출을 요구**하면 성장이 막힐 수 있음.
- **이유:** 수요(견주)는 **공급(인증 보호맘·돌봄 제공자)** 이 쌓인 뒤 따라오는 경우가 많음. 공급 측 비용·허들이 크면 **초기 네트워크가 형성되지 않음**.
- **개선 방향:** 최소 **1~2개 지역(동·구 단위 등)** 에서 **완전 무료**로 점유·신뢰를 쌓는 데 집중. 수익화 로직은 당분간 **프로모션 무료 상태로 락**해 두고, 충분한 공급·반복 이용이 보일 때 단계적으로 검토.

## 코드베이스와의 대응

| 항목 | 구현 |
|------|------|
| 기본 동작 | `VITE_PROMO_FREE_LISTINGS` **미설정·빈 값 = 무료** (`src/lib/promoFlags.ts`) |
| 유료 전환 | 배포에 `VITE_PROMO_FREE_LISTINGS=false`(또는 `0`) + DB RLS 복원 마이그레이션 등 **의도적 2단계** |
| 이용자 메시지 | 인증 돌봄·결제 페이지(`BillingPage`)에서 초기 성장 구간 **노출 무료** 안내 |

## 운영 체크리스트

- [ ] 프로덕션 `.env`에 실수로 `VITE_PROMO_FREE_LISTINGS=false`가 들어가 있지 않은지 배포 전 확인
- [ ] 유료 전환 시: 약관·공지·기존 인증 돌봄 이용자 안내 문구 준비
- [ ] 지역 집중 실험 시: 마케팅·온보딩 카피를 해당 동네에 맞출지 (`03-marketing-promotion.md`와 연계)
